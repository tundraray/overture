# Value Proposition Design

## Value Proposition Canvas (Strategyzer)

Two halves that must achieve **fit**:

### Customer Profile (Right Side)

| Component | Description | Example |
|-----------|-------------|---------|
| **Customer Jobs** | Tasks they're trying to complete (functional, emotional, social) | "Generate monthly reports for the board" |
| **Pains** | Frustrations, risks, obstacles in getting jobs done | "Takes 2 days, error-prone, looks unprofessional" |
| **Gains** | Desired outcomes, benefits they seek | "Save time, look competent, impress stakeholders" |

### Value Map (Left Side)

| Component | Description | Maps To |
|-----------|-------------|---------|
| **Products & Services** | What you offer | Enables customer jobs |
| **Pain Relievers** | How you remove/reduce pains | Maps to customer pains |
| **Gain Creators** | How you create desired outcomes | Maps to customer gains |

### Achieving Fit

```
FIT = Pain Relievers address top pains
    + Gain Creators deliver top gains
    + Products enable important jobs
```

**Three levels of fit**:
1. **Problem-Solution Fit**: Evidence that customers have these jobs/pains/gains
2. **Product-Market Fit**: Evidence that your value map relieves real pains and creates real gains
3. **Business Model Fit**: Evidence that it generates sustainable revenue

## Multi-Stakeholder VPC (B2B)

In B2B, multiple stakeholders have different jobs/pains/gains:

| Stakeholder | Primary Job | Key Pain | Key Gain |
|------------|------------|----------|----------|
| End User | Get work done efficiently | Tool friction, learning curve | Save time, feel productive |
| Evaluator | Find best solution | Risk of wrong choice | Look smart, reduce risk |
| Budget Holder | Control spend | ROI uncertainty | Clear cost justification |
| IT/Security | Maintain standards | Integration risk, security gaps | Compliance, easy admin |

**Rule**: Your value proposition must resonate with ALL decision stakeholders, not just the end user.

## JTBD Integration

Value Proposition Canvas and Jobs-to-Be-Done are complementary:

```
JTBD provides → Customer Jobs (functional, emotional, social)
VPC adds → Pains (obstacles to job completion) + Gains (desired outcomes)
VPC maps → Your solution to those specific jobs/pains/gains
```

If AJTBD plugin is available, feed its jobs graph directly into the Customer Profile side.

## Differentiation Spectrum

| Level | Description | Defensibility |
|-------|-------------|--------------|
| **Parity** | Same as competitors | None — compete on price/distribution |
| **Feature** | Better features on same dimensions | Low — easily copied |
| **Experience** | Fundamentally better user experience | Medium — harder to replicate |
| **Platform** | Network effects, ecosystem, data moats | High — structural advantage |
| **Category** | Redefine the category entirely | Highest — competitors can't compare |

**Goal**: Aim for Experience or higher. Feature-level differentiation is a treadmill.
